== Running the CLI

[source,bash]
----
dart run bin/metaflac.dart <command> [options] <file.flac> [file2.flac ...]
----

Multiple files can be specified; each is processed in order.

== Modern Subcommands

=== inspect

Display a summary of all metadata in a FLAC file.

[source,bash]
----
dart run bin/metaflac.dart inspect song.flac
dart run bin/metaflac.dart inspect --json song.flac
----

=== blocks list

List all metadata blocks with their types and sizes.

[source,bash]
----
dart run bin/metaflac.dart blocks list song.flac
dart run bin/metaflac.dart blocks list --json song.flac
----

=== tags list

List all Vorbis comment tags.

[source,bash]
----
dart run bin/metaflac.dart tags list song.flac
dart run bin/metaflac.dart tags list --json song.flac
----

=== tags set

Set a tag value, replacing any existing values for that key.

[source,bash]
----
dart run bin/metaflac.dart tags set song.flac "ARTIST=New Artist"
dart run bin/metaflac.dart tags set song.flac "ARTIST=New Artist" "ALBUM=New Album"
----

=== tags add

Add a tag value without removing existing values for the key.

[source,bash]
----
dart run bin/metaflac.dart tags add song.flac "GENRE=Rock"
----

=== tags remove

Remove all values for a tag key.

[source,bash]
----
dart run bin/metaflac.dart tags remove song.flac "ARTIST"
----

=== tags clear

Remove all Vorbis comment tags.

[source,bash]
----
dart run bin/metaflac.dart tags clear song.flac
----

=== tags import

Import tags from a text file (one `KEY=VALUE` per line).

[source,bash]
----
dart run bin/metaflac.dart tags import --from=tags.txt song.flac
----

=== tags export

Export tags to stdout or a file.

[source,bash]
----
dart run bin/metaflac.dart tags export song.flac
dart run bin/metaflac.dart tags export --output=tags.txt song.flac
----

=== picture add

Add an image file as an embedded picture block.

[source,bash]
----
dart run bin/metaflac.dart picture add --file=cover.jpg song.flac
----

=== picture remove

Remove picture blocks.

[source,bash]
----
dart run bin/metaflac.dart picture remove --all song.flac
----

=== picture export

Export an embedded picture to a file.

[source,bash]
----
dart run bin/metaflac.dart picture export --output=cover.jpg song.flac
----

=== padding set

Set the padding block to a specific size in bytes.

[source,bash]
----
dart run bin/metaflac.dart padding set song.flac 4096
----

=== padding remove

Remove all padding blocks.

[source,bash]
----
dart run bin/metaflac.dart padding remove song.flac
----

== Compatibility Flags

For users migrating from the reference `metaflac` tool, legacy flags are supported.
See xref:migration.adoc[Migration from metaflac] for a complete mapping and known differences.

=== List and Show Operations

These flags read metadata and print one or more values to stdout.
Each flag exits after producing its output and does not modify the file.

[cols="3,4", options="header"]
|===
| Legacy Flag | Description

| `--list`
| List all metadata blocks; equivalent to `blocks list`

| `--show-md5`
| Print the MD5 signature from STREAMINFO

| `--show-md5sum`
| Alias for `--show-md5`; provided for `metaflac` compatibility

| `--show-min-blocksize`
| Print the minimum block size from STREAMINFO (one integer)

| `--show-max-blocksize`
| Print the maximum block size from STREAMINFO (one integer)

| `--show-min-framesize`
| Print the minimum frame size from STREAMINFO (one integer)

| `--show-max-framesize`
| Print the maximum frame size from STREAMINFO (one integer)

| `--show-sample-rate`
| Print the sample rate from STREAMINFO (one integer)

| `--show-channels`
| Print the channel count from STREAMINFO (one integer)

| `--show-bps`
| Print the bits-per-sample value from STREAMINFO (one integer)

| `--show-total-samples`
| Print the total sample count from STREAMINFO (one integer)

| `--show-vendor-tag`
| Print the Vorbis comment vendor string

| `--show-tag=NAME`
| Print all values for the named tag field; equivalent to `tags list --tag NAME`

| `--show-all-tags`
| Print all Vorbis comment tags; equivalent to `tags list`
|===

=== Tag Write Operations

[cols="3,4", options="header"]
|===
| Legacy Flag | Description

| `--set-tag=KEY=VALUE`
| Set a tag; equivalent to `tags set <file> KEY=VALUE`

| `--remove-tag=KEY`
| Remove all values for a tag key; equivalent to `tags remove <file> KEY`

| `--remove-first-tag=FIELD`
| Remove only the first matching entry for FIELD; equivalent to `tags remove --first <file> FIELD`

| `--remove-all-tags`
| Remove all Vorbis comment tags; equivalent to `tags clear`

| `--remove-all-tags-except=NAME1[=NAME2[=...]]`
| Remove all tags except the named fields. Uses `metaflac`'s `=` separator inside the value (e.g. `--remove-all-tags-except=TITLE=ARTIST=ALBUM`)

| `--remove-replay-gain`
| Remove all `REPLAYGAIN_*` tags (see <<non-goals>>)

| `--set-tag-from-file=FIELD=FILE`
| Read the tag value from FILE and set it; equivalent to `tags set --from-file FIELD=FILE`

| `--export-tags-to=FILE`
| Export Vorbis comments to FILE; use `-` for stdout. Equivalent to `tags export --output=FILE`

| `--import-tags-from=FILE`
| Import tags from FILE (one `KEY=VALUE` per line); equivalent to `tags import --from=FILE`
|===

=== Picture Operations

[cols="3,4", options="header"]
|===
| Legacy Flag | Description

| `--export-picture-to=FILE`
| Export the first embedded picture to FILE; equivalent to `picture export --output=FILE`

| `--import-picture-from=FILE`
| Import an image file as an embedded picture block; equivalent to `picture add --file=FILE`
|===

== Global Options

These options are accepted by both the modern subcommand path and the legacy flag path.

[cols="2,4", options="header"]
|===
| Option | Description

| `--json`
| Output results in JSON format

| `--dry-run`
| Show what changes would be made without writing to the file

| `--continue-on-error`
| Continue processing remaining files if one fails

| `--quiet`, `-q`
| Suppress normal output

| `--preserve-modtime`
| Restore the file's original modification time after writing

| `--with-filename`
| Include the filename as a prefix in output lines

| `--no-filename`
| Suppress the filename prefix even when processing multiple files; overrides `--with-filename`

| `-o FILE`, `--output-name=FILE`
| Write output to FILE rather than modifying the input file in place; the original file is left untouched

| `--no-utf8-convert`
| Accepted for `metaflac` compatibility; has no effect. Dart strings are UTF-16 internally and are converted at serialisation boundaries automatically.

| `--dont-use-padding`
| Disable padding reuse and force a full file rewrite for every change, regardless of whether the modified metadata would fit within the existing padding region
|===

== Exit Codes

[cols="1,3", options="header"]
|===
| Code | Meaning

| 0
| Success

| 1
| General error

| 2
| Invalid arguments

| 3
| Invalid FLAC file

| 4
| I/O error
|===

[#non-goals]
== Non-Goals

=== ReplayGain Analysis (`--add-replay-gain`, `--scan-replay-gain`)

`--add-replay-gain` and `--scan-replay-gain` are permanently out of scope.
Computing ReplayGain values requires decoding and analysing PCM audio frames.
{project-name} is a metadata-only library and does not decode audio data; implementing these flags would require pulling in an audio decoding dependency that is incompatible with the library's design goal of having no native-code or platform-specific dependencies.

See link:https://github.com/bovinemagnet/dart-metaflac/issues/19[issue #19] for the canonical rationale.

NOTE: `--remove-replay-gain` *is* supported because it only removes tag fields by name — no audio decoding is involved.

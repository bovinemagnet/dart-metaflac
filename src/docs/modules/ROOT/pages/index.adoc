= dart_metaflac
Paul Snow
:description: A pure Dart library and CLI tool for reading and writing FLAC audio metadata

include::_attributes.adoc[]

{project-name} is a pure Dart library for reading and writing FLAC audio metadata.
It has no `dart:io` dependency in the core, making it suitable for Dart, Flutter, and web applications.

== Features

* **Parse** FLAC metadata from bytes or streams
* **Read** Vorbis comments, stream info, pictures, padding, and all block types
* **Edit** tags, pictures, and padding with an immutable document model
* **Transform** FLAC files in-memory or via memory-efficient streaming
* **File I/O** with atomic writes (temp file + rename) for safe updates
* **CLI tool** with modern subcommands and `metaflac`-compatible flags
* **Round-trip safe** -- unknown block types are preserved as-is

== Platform Support

The library has two entry points with different platform availability:

* `package:dart_metaflac/dart_metaflac.dart` -- core; no `dart:io`; available on all platforms.
* `package:dart_metaflac/io.dart` -- file/IO layer; requires `dart:io`; not available on Flutter Web.

[cols="1,1,1", options="header"]
|===
| Platform | `dart_metaflac.dart` | `io.dart`

| Dart CLI
| Yes (full support)
| Yes (file I/O + CLI)

| Flutter Mobile
| Yes
| Yes (file I/O)

| Flutter Web
| Yes (bytes and streams)
| No

| Flutter Desktop
| Yes
| Yes (file I/O)
|===

== Module Overview

The library is organised into layered modules, each with a clear responsibility:

[cols="1,2,2", options="header"]
|===
| Module | Responsibility | Key Types

| `model`
| Immutable domain objects
| `FlacMetadataDocument`, `VorbisComments`, `StreamInfoBlock`, `PictureBlock`

| `binary`
| Low-level parsing and serialisation
| `FlacParser`, `FlacSerializer`, `ByteReader`, `ByteWriter`

| `edit`
| Mutation operations
| `FlacMetadataEditor`, `MetadataMutation`, `PaddingStrategy`

| `transform`
| Transform planning and streaming
| `FlacTransformer`, `StreamRewriter`, `FlacTransformPlan`

| `api`
| High-level convenience functions
| `readFlacMetadata`, `readFlacMetadataFromBytes`, `transformFlac`

| `io`
| File persistence (uses `dart:io`)
| `FlacFileEditor`, `AtomicWriter`, `ModTimePreserver`

| `error`
| Typed exception hierarchy
| `FlacMetadataException`, `InvalidFlacException`, `FlacIoException`
|===

See xref:architecture.adoc[Architecture] for a detailed breakdown of the module structure and design decisions.

== Dart API Equivalents

[cols="2,3", options="header"]
|===
| metaflac Operation | Dart API

| Show metadata blocks
| `FlacMetadataDocument.readFromBytes(bytes).blocks`

| Show MD5
| `doc.streamInfo.md5Signature`

| Read a tag
| `doc.vorbisComment?.comments.valuesOf('ARTIST')`

| Set a tag
| `doc.edit((e) => e.setTag('ARTIST', ['value']))`

| Add a tag (multi-value)
| `doc.edit((e) => e.addTag('GENRE', 'Rock'))`

| Remove a tag
| `doc.edit((e) => e.removeTag('COMMENT'))`

| Remove specific value
| `doc.edit((e) => e.removeExactTagValue('GENRE', 'Rock'))`

| Clear all tags
| `doc.edit((e) => e.clearTags())`

| Export tags
| Iterate `doc.vorbisComment?.comments.entries`

| Import tags
| Parse file, then `editor.setTag()` / `editor.addTag()`

| Add picture
| `doc.edit((e) => e.addPicture(pictureBlock))`

| Remove picture by type
| `doc.edit((e) => e.removePictureByType(PictureType.frontCover))`

| Remove all pictures
| `doc.edit((e) => e.removeAllPictures())`

| File read/write
| `FlacFileEditor.readFile(path)` / `FlacFileEditor.updateFile(path, mutations: [...])`
|===

== Usage Examples

=== Reading Tags

**metaflac:**
[source,bash]
----
metaflac --show-tag=ARTIST song.flac
----

**{project-name} CLI:**
[source,bash]
----
dart run bin/metaflac.dart tags list song.flac
----

**{project-name} Dart API:**
[source,dart]
----
final doc = FlacMetadataDocument.readFromBytes(bytes);
final artists = doc.vorbisComment?.comments.valuesOf('ARTIST');
print(artists); // ['Artist Name']
----

=== Setting Tags

**metaflac:**
[source,bash]
----
metaflac --set-tag=ARTIST=NewArtist song.flac
----

**{project-name} CLI:**
[source,bash]
----
dart run bin/metaflac.dart tags set song.flac "ARTIST=NewArtist"
----

**{project-name} Dart API:**
[source,dart]
----
final updated = doc.edit((editor) {
  editor.setTag('ARTIST', ['NewArtist']);
});
final newBytes = updated.toBytes();
----

=== Exporting Tags

**metaflac:**
[source,bash]
----
metaflac --export-tags-to=tags.txt song.flac
----

**{project-name} CLI:**
[source,bash]
----
dart run bin/metaflac.dart tags export --output=tags.txt song.flac
----

=== Adding a Picture

**metaflac:**
[source,bash]
----
metaflac --import-picture-from=cover.jpg song.flac
----

**{project-name} CLI:**
[source,bash]
----
dart run bin/metaflac.dart picture add --file=cover.jpg song.flac
----

== Known Differences

. **Vorbis comment keys** -- {project-name} preserves the original case of keys when stored, but all lookups are case-insensitive (per the Vorbis specification).
The reference `metaflac` also follows this behaviour.

. **Multiple values** -- `setTag` in {project-name} replaces _all_ values for a key.
Use `addTag` to append without removing existing values.
The reference `metaflac` `--set-tag` appends by default.

. **Picture import** -- {project-name} uses `picture add --file` rather than the specification format that encodes type, MIME, description, and dimensions in the argument.
When using the legacy `--import-picture-from` flag, the behaviour matches the reference tool.

. **Output format** -- {project-name} supports `--json` for structured output, which the reference `metaflac` does not.

. **Batch processing** -- {project-name} supports `--continue-on-error` for processing multiple files, continuing even if one file fails.

. **Dry run** -- {project-name} supports `--dry-run` to preview changes without writing, which the reference `metaflac` does not.

. **Padding** -- {project-name} provides explicit padding management via `padding set` and `padding remove` subcommands.
The reference `metaflac` has `--add-padding` and `--remove` but with different semantics.
